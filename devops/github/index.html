GitHub

GitHub is a web-based platform for version control and collaboration using Git.

Key Features

  • Repositories - Store and manage code
  • Branches - Work on features in isolation
  • Pull Requests - Propose and review changes
  • Issues - Track bugs and feature requests
  • Actions - Automate workflows (CI/CD)
  • Projects - Kanban-style project management

GitHub Workflow

1 Fork Repository

Create your own copy of a repository to make changes.

2 Clone Locally

git clone https://github.com/username/repo.git

3 Create Branch

git checkout -b feature/new-feature

4 Make Changes & Commit

git add .
git commit -m "Add new feature"

5 Push to GitHub

git push origin feature/new-feature

6 Create Pull Request

Open a PR on GitHub to merge your changes into the main branch.

Best Practices

  • Write meaningful commit messages
  • Use branch protection rules
  • Require pull request reviews
  • Use issue templates
  • Add a README and LICENSE
  • Use .gitignore to exclude files
