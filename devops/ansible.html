Ansible

Ansible is an agentless automation tool for configuration management, application deployment, and task automation.

Ansible Commands

CommandDescription
ansible all -m pingPing all hosts
ansible-playbook playbook.ymlRun playbook
ansible-playbook -i inventory playbook.ymlSpecify inventory
ansible-playbook --check playbook.ymlDry run
ansible-playbook -e "var=value" playbook.ymlExtra variables
ansible-galaxy install roleInstall role
ansible-vault encrypt fileEncrypt file
ansible-vault decrypt fileDecrypt file

Inventory

# inventory.ini
[webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com ansible_user=admin

[all:vars]
ansible_python_interpreter=/usr/bin/python3

Playbook Example

# playbook.yml
---
- name: Configure web servers
  hosts: webservers
  become: yes
  vars:
    http_port: 80

  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present
        update_cache: yes

    - name: Start nginx
      service:
        name: nginx
        state: started
        enabled: yes

    - name: Copy config file
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf
      notify: Restart nginx

  handlers:
    - name: Restart nginx
      service:
        name: nginx
        state: restarted

Variables

# vars/main.yml
http_port: 80
max_clients: 200

# Using variables
- name: Configure port
  template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
  vars:
    port: "{{ http_port }}"

# Jinja2 template
listen {{ http_port }};
worker_connections {{ max_clients }};

Conditionals and Loops

# Conditional
- name: Install on Debian
  apt:
    name: nginx
  when: ansible_os_family == "Debian"

# Loop
- name: Install packages
  apt:
    name: "{{ item }}"
    state: present
  loop:
    - nginx
    - git
    - curl
Terraform vs Ansible

Terraform: Best for provisioning infrastructure (VMs, networks, cloud resources).
Ansible: Best for configuring servers and deploying applications.
