Volumes & Storage

Storage Types

TypeDescriptionUse Case
VolumesManaged by DockerProduction data persistence
Bind MountsHost filesystem pathDevelopment, config files
tmpfsMemory onlySensitive data, temp files

Volume Commands

CommandDescription
docker volume lsList volumes
docker volume create nameCreate volume
docker volume inspect nameView volume details
docker volume rm nameRemove volume
docker volume pruneRemove unused volumes

Using Volumes

# Create named volume
docker volume create mydata

# Run with volume
docker run -v mydata:/app/data nginx

# Using --mount (more explicit)
docker run --mount source=mydata,target=/app/data nginx

# Anonymous volume
docker run -v /app/data nginx

Bind Mounts

# Mount host directory
docker run -v /host/path:/container/path nginx

# Mount current directory
docker run -v $(pwd):/app nginx

# Read-only mount
docker run -v $(pwd):/app:ro nginx

# Using --mount
docker run --mount type=bind,source=/host/path,target=/container/path nginx

Volumes in Docker Compose

version: '3.8'

services:
  db:
    image: postgres
    volumes:
      # Named volume
      - postgres_data:/var/lib/postgresql/data
      # Bind mount
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
      # Anonymous volume (for node_modules etc)
      - /var/lib/postgresql/data

  web:
    build: .
    volumes:
      - ./src:/app/src:ro
      - node_modules:/app/node_modules

volumes:
  postgres_data:
  node_modules:

Backup and Restore

# Backup volume to tar
docker run --rm -v mydata:/data -v $(pwd):/backup \
  ubuntu tar cvf /backup/backup.tar /data

# Restore from tar
docker run --rm -v mydata:/data -v $(pwd):/backup \
  ubuntu tar xvf /backup/backup.tar -C /
Data Persistence

Container filesystems are ephemeral. Always use volumes for data that needs to persist beyond the container lifecycle.
