Docker Core Concepts

Images

A Docker image is a read-only template containing instructions for creating a container. Images are built in layers.

CommandDescription
docker imagesList images
docker pull nginx:latestDownload image
docker build -t myapp:1.0 .Build image from Dockerfile
docker tag image:tag new:tagTag image
docker push image:tagPush to registry
docker rmi imageRemove image
docker image pruneRemove unused images

Containers

A container is a runnable instance of an image. Containers are isolated but share the host OS kernel.

CommandDescription
docker run imageCreate and start container
docker run -d imageRun in background (detached)
docker run -it image bashInteractive terminal
docker run -p 8080:80 imageMap ports
docker run --name myapp imageName container
docker run -e VAR=value imageSet environment variable
docker psList running containers
docker ps -aList all containers
docker stop containerStop container
docker start containerStart stopped container
docker restart containerRestart container
docker rm containerRemove container
docker rm -f containerForce remove running container

Interacting with Containers

CommandDescription
docker exec -it container bashExecute command in container
docker logs containerView logs
docker logs -f containerFollow logs
docker inspect containerView container details
docker cp file container:/pathCopy file to container
docker cp container:/path fileCopy file from container
docker statsView resource usage
docker top containerView running processes

Registries

A registry stores Docker images. Docker Hub is the default public registry.

Docker HubDefault public registry

Azure Container RegistryAzure container registry
 
# Login to registry
docker login

# Pull from registry
docker pull nginx:latest

# Tag for push
docker tag myapp:latest username/myapp:latest

# Push to registry
docker push username/myapp:latest

Image Layers

Images are built in layers. Each instruction in a Dockerfile creates a layer. Layers are cached and reused.

# Each instruction creates a layer
FROM ubuntu:22.04          # Base layer
RUN apt-get update         # Layer 2
RUN apt-get install nginx  # Layer 3
COPY app /var/www          # Layer 4
CMD ["nginx", "-g", "daemon off;"]
Layer Caching

Docker caches layers. If a layer hasn't changed, it's reused from cache. Order your Dockerfile with least-changing instructions first for faster builds.
