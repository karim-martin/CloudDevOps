Git Version Control

Configuration

CommandDescription
git config --global user.name "Name"Set username
git config --global user.email "email"Set email
git config --listView all settings

Basic Commands

CommandDescription
git initInitialize new repository
git clone urlClone repository
git statusCheck status
git add fileStage file
git add .Stage all changes
git commit -m "msg"Commit with message
git commit -am "msg"Add and commit tracked files
git logView commit history
git log --onelineCompact history
git diffShow unstaged changes
git diff --stagedShow staged changes

Branching

CommandDescription
git branchList branches
git branch nameCreate branch
git checkout nameSwitch to branch
git checkout -b nameCreate and switch
git switch nameSwitch branch (newer)
git switch -c nameCreate and switch (newer)
git merge nameMerge branch into current
git branch -d nameDelete merged branch
git branch -D nameForce delete branch

Remote Operations

CommandDescription
git remote -vView remotes
git remote add origin urlAdd remote
git push origin branchPush to remote
git push -u origin branchPush and set upstream
git pullFetch and merge
git fetchFetch without merge
git push --forceForce push (careful!)

Undoing Changes

CommandDescription
git checkout -- fileDiscard changes in file
git restore fileDiscard changes (newer)
git reset HEAD fileUnstage file
git restore --staged fileUnstage file (newer)
git reset --soft HEAD~1Undo last commit, keep changes
git reset --hard HEAD~1Undo last commit, discard changes
git revert commitCreate undo commit

Stashing

CommandDescription
git stashStash changes
git stash save "msg"Stash with message
git stash listList stashes
git stash popApply and remove stash
git stash applyApply stash, keep it
git stash dropDelete stash

Common Workflow

# Start new feature
git checkout main
git pull origin main
git checkout -b feature/my-feature

# Work on feature
git add .
git commit -m "Add feature"

# Push to remote
git push -u origin feature/my-feature

# After PR is merged
git checkout main
git pull origin main
git branch -d feature/my-feature
Best Practices
  • Write clear, descriptive commit messages
  • Commit often, push regularly
  • Use branches for new features
  • Pull before pushing to avoid conflicts
  • Never commit sensitive data
