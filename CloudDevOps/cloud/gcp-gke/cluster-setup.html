GKE Cluster Setup

Create Autopilot Cluster

Autopilot is the simplest option - Google manages everything.

# Create Autopilot cluster
gcloud container clusters create-auto my-cluster \
    --region=us-central1 \
    --project=YOUR_PROJECT_ID

Create Standard Cluster

# Create Standard cluster
gcloud container clusters create my-cluster \
    --zone=us-central1-a \
    --num-nodes=3 \
    --machine-type=e2-medium \
    --enable-autoscaling \
    --min-nodes=1 \
    --max-nodes=5 \
    --project=YOUR_PROJECT_ID

Connect to Cluster

# Get credentials
gcloud container clusters get-credentials my-cluster \
    --zone=us-central1-a \
    --project=YOUR_PROJECT_ID

# Verify connection
kubectl get nodes
kubectl cluster-info

Node Pools

# Create node pool
gcloud container node-pools create high-memory \
    --cluster=my-cluster \
    --zone=us-central1-a \
    --machine-type=e2-highmem-4 \
    --num-nodes=2

# List node pools
gcloud container node-pools list --cluster=my-cluster --zone=us-central1-a

# Resize node pool
gcloud container clusters resize my-cluster \
    --zone=us-central1-a \
    --node-pool=default-pool \
    --num-nodes=5

Common Cluster Options

FlagDescription
--zoneZonal cluster (single zone)
--regionRegional cluster (multi-zone)
--machine-typeVM type for nodes
--num-nodesNodes per zone
--enable-autoscalingEnable node autoscaling
--enable-ip-aliasVPC-native cluster
--enable-private-nodesPrivate cluster
--workload-poolEnable Workload Identity

Recommended Production Setup

gcloud container clusters create prod-cluster \
    --region=us-central1 \
    --num-nodes=2 \
    --machine-type=e2-standard-4 \
    --enable-autoscaling \
    --min-nodes=1 \
    --max-nodes=10 \
    --enable-ip-alias \
    --enable-autorepair \
    --enable-autoupgrade \
    --workload-pool=PROJECT_ID.svc.id.goog \
    --release-channel=regular

Manage Cluster

CommandDescription
gcloud container clusters listList clusters
gcloud container clusters describe nameCluster details
gcloud container clusters update nameUpdate cluster
gcloud container clusters delete nameDelete cluster
Cost Warning

GKE clusters incur costs even when idle. Delete unused clusters to avoid charges.
