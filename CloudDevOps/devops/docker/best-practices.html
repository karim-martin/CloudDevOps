Docker Best Practices

Reduce Image Size

  • Use minimal base images (alpine, slim, distroless)
  • Use multi-stage builds
  • Combine RUN commands to reduce layers
  • Remove unnecessary files after installation
  • Use .dockerignore
# Bad - multiple layers
RUN apt-get update
RUN apt-get install -y curl
RUN apt-get clean

# Good - single layer, cleanup
RUN apt-get update && \
    apt-get install -y --no-install-recommends curl && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

Security

  • Don't run as root
  • Use specific image tags, not latest
  • Scan images for vulnerabilities
  • Don't store secrets in images
  • Use read-only filesystems when possible
# Create non-root user
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

# Use specific version
FROM node:20.10.0-alpine

# Read-only container
docker run --read-only --tmpfs /tmp myapp

Optimize Build Cache

# Good - copy dependency files first
COPY package*.json ./
RUN npm ci

# Then copy source (changes more often)
COPY . .

Health Checks

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1

Logging

  • Log to stdout/stderr, not files
  • Use structured logging (JSON)
  • Configure log rotation
# Configure logging driver
docker run --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  myapp

Production Checklist

CategoryCheck
ImageUse specific tags, minimal base image
SecurityNon-root user, no secrets in image
ResourcesSet memory and CPU limits

LoggingStdout/stderr, rotation configured

StorageVolumes for persistent data
 
# Production-ready docker-compose
services:
  app:
    image: myapp:1.2.3
    user: "1000:1000"
    read_only: true
    restart: unless-stopped
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
