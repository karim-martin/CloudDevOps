Operations

Automate deployments with CI/CD pipelines, scale your application based on demand, and monitor health and performance with Cloud Monitoring.

What You'll Configure

CI/CD

GitHub Actions pipeline with Workload Identity Federation for keyless authentication.

Scaling

Horizontal Pod Autoscaler and GKE Cluster Autoscaler for automatic scaling.

Monitoring

Cloud Monitoring dashboards, alerts, and log-based metrics.

Quick Operations Commands

# View deployment status
kubectl rollout status deployment/dotnet-api -n app

# View rollout history
kubectl rollout history deployment/dotnet-api -n app

# Rollback to previous version
kubectl rollout undo deployment/dotnet-api -n app

# Scale deployment manually
kubectl scale deployment/dotnet-api --replicas=5 -n app

# View HPA status
kubectl get hpa -n app

# View pod resource usage
kubectl top pods -n app

# View cluster node usage
kubectl top nodes

# View recent events
kubectl get events -n app --sort-by='.lastTimestamp' | tail -20

# Stream logs from all pods
kubectl logs -l app=dotnet-api -n app -f --tail=100

GitOps Alternative

For more advanced deployments, consider GitOps tools like ArgoCD or Flux. They continuously sync your cluster state with Git, providing audit trails and easier rollbacks.

Resource Requests

HPA requires resource requests defined in your Deployment. Without CPU/memory requests, HPA cannot calculate utilization percentages for scaling decisions.
