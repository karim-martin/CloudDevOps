Cloud Storage Setup

Configure Cloud Storage for file uploads, user content, and application assets.

Storage Options

OptionBest ForAccess Pattern
Cloud StorageFiles, images, backups, static assetsAPI-based (HTTP)
FilestoreShared filesystem (NFS)File system mount

For most web applications, Cloud Storage is recommended. Use Filestore only if you need traditional file system access.

Create Cloud Storage Bucket

# Set bucket name (must be globally unique)
export BUCKET_NAME="${PROJECT_ID}-app-uploads"

# Create bucket
gsutil mb -l $REGION -c STANDARD gs://$BUCKET_NAME

# Verify bucket
gsutil ls gs://$BUCKET_NAME

Storage Classes

ClassUse CaseMin Storage
STANDARDFrequently accessed dataNone
NEARLINEAccessed < 1/month30 days
COLDLINEAccessed < 1/quarter90 days
ARCHIVEAccessed < 1/year365 days

Configure Bucket IAM

# Grant the app service account access to the bucket
gsutil iam ch \
    serviceAccount:app-workload-sa@${PROJECT_ID}.iam.gserviceaccount.com:objectAdmin \
    gs://$BUCKET_NAME

# Verify permissions
gsutil iam get gs://$BUCKET_NAME

Configure CORS (for Direct Upload)

If your frontend uploads directly to Cloud Storage:

// cors-config.json
[
  {
    "origin": ["https://app.example.com", "http://localhost:3000"],
    "method": ["GET", "POST", "PUT", "DELETE", "HEAD"],
    "responseHeader": ["Content-Type", "Content-Length", "Content-Range"],
    "maxAgeSeconds": 3600
  }
]
# Apply CORS configuration
gsutil cors set cors-config.json gs://$BUCKET_NAME

# Verify CORS
gsutil cors get gs://$BUCKET_NAME

.NET Integration

Install NuGet Package

# In your .NET project
dotnet add package Google.Cloud.Storage.V1

Upload Files from .NET

using Google.Cloud.Storage.V1;

public class StorageService
{
    private readonly StorageClient _storageClient;
    private readonly string _bucketName;

    public StorageService(string bucketName)
    {
        // Uses Workload Identity automatically when running in GKE
        _storageClient = StorageClient.Create();
        _bucketName = bucketName;
    }

    public async Task<string> UploadFileAsync(Stream fileStream, string fileName, string contentType)
    {
        var objectName = $"uploads/{DateTime.UtcNow:yyyy/MM/dd}/{Guid.NewGuid()}/{fileName}";

        await _storageClient.UploadObjectAsync(
            _bucketName,
            objectName,
            contentType,
            fileStream
        );

        return $"gs://{_bucketName}/{objectName}";
    }

    public async Task<Stream> DownloadFileAsync(string objectName)
    {
        var stream = new MemoryStream();
        await _storageClient.DownloadObjectAsync(_bucketName, objectName, stream);
        stream.Position = 0;
        return stream;
    }

    public string GetPublicUrl(string objectName)
    {
        return $"https://storage.googleapis.com/{_bucketName}/{objectName}";
    }
}

Register Service in Program.cs

// Program.cs
var bucketName = Environment.GetEnvironmentVariable("GCS_BUCKET")
    ?? throw new InvalidOperationException("GCS_BUCKET not set");

builder.Services.AddSingleton(new StorageService(bucketName));

Kubernetes Configuration

# Pass bucket name as environment variable in deployment
env:
- name: GCS_BUCKET
  value: "my-project-app-uploads"

Generate Signed URLs

For secure, time-limited access to private files:

public string GenerateSignedUrl(string objectName, TimeSpan duration)
{
    var urlSigner = UrlSigner.FromServiceAccountCredential(
        GoogleCredential.GetApplicationDefault()
            .UnderlyingCredential as ServiceAccountCredential);

    return urlSigner.Sign(
        _bucketName,
        objectName,
        duration,
        HttpMethod.Get
    );
}

Lifecycle Policy (Optional)

Auto-delete old files or move to cheaper storage:

// lifecycle-config.json
{
  "rule": [
    {
      "action": {"type": "Delete"},
      "condition": {
        "age": 365,
        "matchesPrefix": ["temp/"]
      }
    },
    {
      "action": {"type": "SetStorageClass", "storageClass": "NEARLINE"},
      "condition": {
        "age": 30,
        "matchesPrefix": ["uploads/"]
      }
    }
  ]
}
# Apply lifecycle policy
gsutil lifecycle set lifecycle-config.json gs://$BUCKET_NAME

Workload Identity Integration

When running in GKE with Workload Identity configured, the Google Cloud client libraries automatically use the pod's service account. No credential files needed!
