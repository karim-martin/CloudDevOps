GKE Cluster Setup

Create a private GKE cluster with VPC-native networking and Workload Identity for secure, keyless authentication.

Cluster Type Decision

FeatureStandardAutopilot
Node ManagementYou manageGoogle manages
PricingPer nodePer pod resources
CustomizationFull controlLimited
Best ForProduction, custom needsSimplicity, variable loads

This guide uses Standard mode for full control. For simpler setups, consider Autopilot.

Create GKE Cluster

# Create private GKE cluster with Workload Identity
gcloud container clusters create app-cluster \
    --region=$REGION \
    --num-nodes=1 \
    --machine-type=e2-medium \
    --disk-size=50GB \
    --network=app-vpc \
    --subnetwork=app-subnet \
    --cluster-secondary-range-name=pods \
    --services-secondary-range-name=services \
    --enable-ip-alias \
    --enable-private-nodes \
    --master-ipv4-cidr=172.16.0.0/28 \
    --enable-master-authorized-networks \
    --master-authorized-networks=0.0.0.0/0 \
    --workload-pool=${PROJECT_ID}.svc.id.goog \
    --enable-autoscaling \
    --min-nodes=1 \
    --max-nodes=5 \
    --tags=gke-node

# This takes 5-10 minutes...

Cluster Configuration Explained

FlagPurpose
--regionRegional cluster (high availability across zones)
--num-nodes=11 node per zone (3 total for regional)
--enable-private-nodesNodes have no public IPs (more secure)
--master-ipv4-cidrPrivate IP range for control plane
--workload-poolEnables Workload Identity
--enable-autoscalingAuto-scale nodes based on demand

Connect kubectl

# Get cluster credentials
gcloud container clusters get-credentials app-cluster --region=$REGION

# Verify connection
kubectl cluster-info

# Check nodes
kubectl get nodes

# Check namespaces
kubectl get namespaces

Create Application Namespace

# Create dedicated namespace for the application
kubectl create namespace app

# Set as default namespace (optional)
kubectl config set-context --current --namespace=app

# Verify
kubectl get namespace app

Create Kubernetes Service Account

Create a Kubernetes Service Account that will be linked to the GCP Service Account via Workload Identity:

# k8s/service-account.yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: app-ksa
  namespace: app
  annotations:
    iam.gke.io/gcp-service-account: app-workload-sa@PROJECT_ID.iam.gserviceaccount.com
# Apply the service account (replace PROJECT_ID first)
sed "s/PROJECT_ID/$PROJECT_ID/g" k8s/service-account.yaml | kubectl apply -f -

# Or create directly
kubectl create serviceaccount app-ksa -n app

# Add annotation for Workload Identity
kubectl annotate serviceaccount app-ksa \
    --namespace app \
    iam.gke.io/gcp-service-account=app-workload-sa@${PROJECT_ID}.iam.gserviceaccount.com

Enable Workload Identity Binding

# Allow KSA to impersonate GSA
gcloud iam service-accounts add-iam-policy-binding \
    app-workload-sa@${PROJECT_ID}.iam.gserviceaccount.com \
    --role="roles/iam.workloadIdentityUser" \
    --member="serviceAccount:${PROJECT_ID}.svc.id.goog[app/app-ksa]"

Verify Workload Identity

# Deploy a test pod to verify Workload Identity works
kubectl run workload-identity-test \
    --image=google/cloud-sdk:slim \
    --namespace=app \
    --serviceaccount=app-ksa \
    --rm -it --restart=Never \
    -- gcloud auth list

# Should show: app-workload-sa@PROJECT_ID.iam.gserviceaccount.com

Alternative: Autopilot Cluster

For a simpler setup with less management overhead:

# Create Autopilot cluster (Google manages everything)
gcloud container clusters create-auto app-cluster-autopilot \
    --region=$REGION \
    --network=app-vpc \
    --subnetwork=app-subnet \
    --cluster-secondary-range-name=pods \
    --services-secondary-range-name=services
Cluster Creation Time

Cluster creation takes 5-10 minutes. You can proceed to create Cloud SQL in a separate terminal while waiting.

Private Cluster Access

With --enable-private-nodes, nodes have no public IPs. They access the internet via Cloud NAT (configured earlier). This is more secure but requires proper NAT setup.
