Cloud SQL Database

Set up a managed Cloud SQL database with private IP connectivity for your .NET 8 application.

Database Choice

DatabaseBest For.NET Package
MySQL 8.0Open source, cost-effectiveMySql.EntityFrameworkCore
SQL ServerEnterprise .NET apps, familiar to .NET devsMicrosoft.EntityFrameworkCore.SqlServer

Option 1: MySQL Setup

# Create MySQL instance with private IP
gcloud sql instances create app-db-mysql \
    --database-version=MYSQL_8_0 \
    --tier=db-custom-2-4096 \
    --region=$REGION \
    --network=projects/$PROJECT_ID/global/networks/app-vpc \
    --no-assign-ip \
    --storage-size=20GB \
    --storage-auto-increase \
    --backup-start-time=02:00 \
    --maintenance-window-day=SUN \
    --maintenance-window-hour=03

# This takes 10-15 minutes...

Create Database and User

# Create database
gcloud sql databases create appdb --instance=app-db-mysql

# Generate secure password
export DB_PASSWORD=$(openssl rand -base64 24)
echo "Save this password: $DB_PASSWORD"

# Create user
gcloud sql users create appuser \
    --instance=app-db-mysql \
    --password=$DB_PASSWORD

# Get private IP
export DB_HOST=$(gcloud sql instances describe app-db-mysql \
    --format="get(ipAddresses[0].ipAddress)")
echo "Database IP: $DB_HOST"

MySQL Connection String for .NET

// appsettings.json format
"ConnectionStrings": {
    "DefaultConnection": "Server=DB_HOST;Database=appdb;User=appuser;Password=DB_PASSWORD;SslMode=Required;"
}

// Environment variable format
Server=${DB_HOST};Database=appdb;User=appuser;Password=${DB_PASSWORD};SslMode=Required;

Option 2: SQL Server Setup

# Create SQL Server instance with private IP
gcloud sql instances create app-db-sqlserver \
    --database-version=SQLSERVER_2019_STANDARD \
    --tier=db-custom-2-4096 \
    --region=$REGION \
    --network=projects/$PROJECT_ID/global/networks/app-vpc \
    --no-assign-ip \
    --storage-size=20GB \
    --storage-auto-increase \
    --root-password=$(openssl rand -base64 24)

# This takes 10-15 minutes...

Create Database and User

# Create database
gcloud sql databases create appdb --instance=app-db-sqlserver

# Generate secure password
export DB_PASSWORD=$(openssl rand -base64 24)
echo "Save this password: $DB_PASSWORD"

# Create user (SQL Server uses different syntax)
gcloud sql users create appuser \
    --instance=app-db-sqlserver \
    --password=$DB_PASSWORD

# Get private IP
export DB_HOST=$(gcloud sql instances describe app-db-sqlserver \
    --format="get(ipAddresses[0].ipAddress)")
echo "Database IP: $DB_HOST"

SQL Server Connection String for .NET

// appsettings.json format
"ConnectionStrings": {
    "DefaultConnection": "Server=DB_HOST;Database=appdb;User Id=appuser;Password=DB_PASSWORD;TrustServerCertificate=True;"
}

// Environment variable format
Server=${DB_HOST};Database=appdb;User Id=appuser;Password=${DB_PASSWORD};TrustServerCertificate=True;

Create Kubernetes Secret

Store the connection string as a Kubernetes secret:

# For MySQL
kubectl create secret generic db-credentials \
    --namespace=app \
    --from-literal=connection-string="Server=${DB_HOST};Database=appdb;User=appuser;Password=${DB_PASSWORD};SslMode=Required;"

# For SQL Server
kubectl create secret generic db-credentials \
    --namespace=app \
    --from-literal=connection-string="Server=${DB_HOST};Database=appdb;User Id=appuser;Password=${DB_PASSWORD};TrustServerCertificate=True;"

# Verify secret created
kubectl get secret db-credentials -n app

Alternative: YAML Secret Definition

# k8s/secrets/db-credentials.yaml
apiVersion: v1
kind: Secret
metadata:
  name: db-credentials
  namespace: app
type: Opaque
stringData:
  connection-string: "Server=10.x.x.x;Database=appdb;User=appuser;Password=SECURE_PASSWORD;SslMode=Required;"
Never Commit Secrets

Don't commit secrets.yaml with real passwords to Git. Use kubectl create secret or external secret management (like Google Secret Manager).

Verify Database Connection

# Test connection from a pod in the cluster
kubectl run db-test \
    --image=mysql:8.0 \
    --namespace=app \
    --rm -it --restart=Never \
    -- mysql -h $DB_HOST -u appuser -p$DB_PASSWORD -e "SELECT 1;"

# For SQL Server
kubectl run db-test \
    --image=mcr.microsoft.com/mssql-tools \
    --namespace=app \
    --rm -it --restart=Never \
    -- /opt/mssql-tools/bin/sqlcmd -S $DB_HOST -U appuser -P $DB_PASSWORD -Q "SELECT 1"

Cloud SQL Commands Reference

CommandDescription
gcloud sql instances listList all instances
gcloud sql instances describe INSTANCEInstance details
gcloud sql databases list --instance=INSTANCEList databases
gcloud sql users list --instance=INSTANCEList users
gcloud sql connect INSTANCE --user=USERConnect via Cloud SQL Proxy

High Availability (Production)

For production, enable high availability:

# Create with high availability
gcloud sql instances create app-db-mysql \
    --database-version=MYSQL_8_0 \
    --tier=db-custom-2-4096 \
    --region=$REGION \
    --availability-type=REGIONAL \
    --network=projects/$PROJECT_ID/global/networks/app-vpc \
    --no-assign-ip

# Or patch existing instance
gcloud sql instances patch app-db-mysql \
    --availability-type=REGIONAL
Private IP Benefits

Using private IP (--no-assign-ip) means traffic stays within Google's network, providing better security and lower latency. No Cloud SQL Proxy needed when connecting from GKE in the same VPC.
