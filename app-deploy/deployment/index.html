Application Deployment

Containerize your React frontend and .NET 8 backend, then deploy them to the GKE cluster.

Deployment Steps

1. Containerize

Create Dockerfiles and build images for both applications.

2. React Frontend

Deploy the React app with ConfigMap and Service.

3. .NET Backend

Deploy the API with Secrets and database connection.

Key Kubernetes Resources

ResourcePurpose
DeploymentManages pod replicas, rolling updates
ServiceInternal load balancing, DNS name
ConfigMapNon-sensitive configuration
SecretSensitive data (passwords, keys)
ServiceAccountPod identity for GCP access

File Structure

k8s/
├── namespace.yaml           # app namespace
├── service-account.yaml     # Workload Identity SA
├── frontend/
│   ├── deployment.yaml      # React deployment
│   ├── service.yaml         # React service
│   └── configmap.yaml       # React config
└── backend/
    ├── deployment.yaml      # .NET deployment
    ├── service.yaml         # .NET service
    └── secret.yaml          # Database credentials
Deployment Order

Deploy in this order: 1) Namespace & ServiceAccount, 2) Secrets & ConfigMaps, 3) Backend deployment, 4) Frontend deployment. This ensures dependencies are ready.
