Security

Secure your GKE application with Workload Identity for GCP service access, Cloud Armor for WAF protection, and Identity-Aware Proxy for authenticated access.

What You'll Configure

Workload Identity

Allow pods to access GCP services without storing service account keys.

Cloud Armor & IAP

Protect with WAF rules, rate limiting, and Google authentication.

Security Best Practices

PracticeImplementation
Least PrivilegeGrant minimal IAM roles to service accounts
No Service Account KeysUse Workload Identity instead of JSON keys
Private NodesGKE nodes without public IPs, use Cloud NAT
Network PoliciesRestrict pod-to-pod communication
Secret ManagementUse Secret Manager instead of K8s Secrets
Image SecurityScan images, use trusted base images
HTTPS OnlyRedirect HTTP to HTTPS, use HSTS
Regular UpdatesEnable auto-upgrade for GKE nodes

Quick Security Checklist

# Check Workload Identity is enabled
gcloud container clusters describe app-cluster \
    --region=us-central1 \
    --format='value(workloadIdentityConfig.workloadPool)'

# Verify KSA to GSA binding
kubectl get serviceaccount app-ksa -n app -o yaml | grep iam.gke.io/gcp-service-account

# Check Cloud Armor policy is attached
gcloud compute backend-services describe BACKEND_NAME --global \
    --format='value(securityPolicy)'

# Verify SSL/TLS configuration
gcloud compute ssl-policies describe app-ssl-policy

# Check node pool has no external IPs
gcloud container node-pools describe default-pool \
    --cluster=app-cluster \
    --region=us-central1 \
    --format='value(config.oauthScopes)'

Network Policies (Optional)

Restrict pod-to-pod traffic within the cluster.

# k8s/security/network-policy.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: backend-policy
  namespace: app
spec:
  podSelector:
    matchLabels:
      app: dotnet-api
  policyTypes:
  - Ingress
  ingress:
  # Allow traffic from frontend pods
  - from:
    - podSelector:
        matchLabels:
          app: react-frontend
    ports:
    - protocol: TCP
      port: 8080
  # Allow traffic from Ingress controller
  - from:
    - namespaceSelector:
        matchLabels:
          name: kube-system
    ports:
    - protocol: TCP
      port: 8080
Defense in Depth

Security is layered. Cloud Armor protects at the edge, Workload Identity secures service access, and Network Policies control internal traffic. Each layer adds protection even if another fails.

Secrets Management

Kubernetes Secrets are base64 encoded, not encrypted. For production, consider using Google Secret Manager with Workload Identity to access secrets securely.
