Prerequisites

Before starting, ensure you have the required accounts, tools, and foundational knowledge.

Required Accounts

AccountPurposeLink
Google CloudGKE, Cloud SQL, Storagecloud.google.com
GitHubSource code, CI/CDgithub.com
Free Credits

New Google Cloud accounts receive $300 in free credits valid for 90 days. This is enough to complete this entire guide multiple times.

For React Development

# Install Node.js (LTS version recommended)
# Using nvm (Node Version Manager):
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install --lts
nvm use --lts

# Verify installation
node --version   # Should be v18+ or v20+
npm --version

For .NET Development

# Install .NET 8 SDK
# Ubuntu/Debian:
wget https://packages.microsoft.com/config/ubuntu/22.04/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
sudo apt update
sudo apt install -y dotnet-sdk-8.0

# macOS:
brew install dotnet@8

# Verify installation
dotnet --version   # Should be 8.x

Verify All Tools

# Run these commands to verify everything is installed
gcloud version
kubectl version --client
docker --version
git --version
node --version
dotnet --version

# Authenticate with Google Cloud
gcloud auth login
gcloud auth application-default login

Sample Applications

You can follow along with your own React and .NET applications, or use these sample projects:

React Frontend (Sample)

# Create a new React app
npx create-react-app my-frontend
cd my-frontend

# Or clone a sample
git clone https://github.com/your-org/sample-react-app.git

.NET 8 Backend (Sample)

# Create a new .NET Web API
dotnet new webapi -n MyApi
cd MyApi

# Or clone a sample
git clone https://github.com/your-org/sample-dotnet-api.git

Minimum Application Requirements

  • React app - Should build to static files (npm run build)
  • .NET app - Web API with health check endpoint (/health or /healthz)
  • Environment variables - Both apps should read config from environment variables
  • Database connection - .NET app should use connection string from environment variable

Recommended Project Structure
