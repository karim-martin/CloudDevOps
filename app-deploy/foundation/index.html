Foundation

Before deploying applications, you need to set up the GCP foundation: a project, proper IAM permissions, and networking infrastructure.

What You'll Configure

Project Setup

Create a GCP project, enable APIs, and configure IAM service accounts.

Networking

Create a VPC, subnets, firewall rules, and Cloud NAT for private nodes.

Foundation Architecture

┌────────────────────────────────────────────────────────────┐
│                      GCP Project                           │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                        VPC                           │  │
│  │  ┌─────────────────┐    ┌─────────────────────────┐  │  │
│  │  │  Public Subnet  │    │     Private Subnet      │  │  │
│  │  │  (Load Balancer)│    │  (GKE Nodes, Cloud SQL) │  │  │
│  │  └─────────────────┘    └─────────────────────────┘  │  │
│  │                              │                       │  │
│  │                         Cloud NAT                    │  │
│  │                      (Outbound Internet)             │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│  IAM: Service Accounts, Roles, Workload Identity           │
└────────────────────────────────────────────────────────────┘

Key Concepts

ComponentPurpose
GCP ProjectIsolated container for all resources and billing
APIsMust be enabled before using GCP services
Service AccountsIdentity for applications and automation
IAM RolesPermissions granted to users and service accounts
VPCVirtual network isolating your resources
SubnetsIP ranges within your VPC for different purposes
Firewall RulesControl traffic in and out of your network
Cloud NATAllows private resources to access the internet
Why This Matters

A properly configured foundation ensures security, network isolation, and least-privilege access. Skipping these steps often leads to security issues and difficult troubleshooting later.
