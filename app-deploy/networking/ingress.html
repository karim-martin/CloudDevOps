Ingress

Configure GKE Ingress to expose your application with a Google Cloud Load Balancer, path-based routing, and static IP address.

BackendConfig

Configure health checks and timeouts for the load balancer backends.

# k8s/networking/backend-config.yaml
apiVersion: cloud.google.com/v1
kind: BackendConfig
metadata:
  name: app-backend-config
  namespace: app
spec:
  healthCheck:
    checkIntervalSec: 15
    timeoutSec: 5
    healthyThreshold: 2
    unhealthyThreshold: 3
    type: HTTP
    requestPath: /health
    port: 8080
  timeoutSec: 30
  connectionDraining:
    drainingTimeoutSec: 60
  logging:
    enable: true
    sampleRate: 1.0

Update Services with BackendConfig

Add BackendConfig annotation to your services.

# Update k8s/frontend/service.yaml
apiVersion: v1
kind: Service
metadata:
  name: react-frontend
  namespace: app
  labels:
    app: react-frontend
  annotations:
    cloud.google.com/backend-config: '{"default": "app-backend-config"}'
    cloud.google.com/neg: '{"ingress": true}'  # Network Endpoint Groups
spec:
  type: ClusterIP
  selector:
    app: react-frontend
  ports:
  - port: 80
    targetPort: 80
    name: http
---
# Update k8s/backend/service.yaml
apiVersion: v1
kind: Service
metadata:
  name: dotnet-api
  namespace: app
  labels:
    app: dotnet-api
  annotations:
    cloud.google.com/backend-config: '{"default": "app-backend-config"}'
    cloud.google.com/neg: '{"ingress": true}'
spec:
  type: ClusterIP
  selector:
    app: dotnet-api
  ports:
  - port: 8080
    targetPort: 8080
    name: http

Ingress Resource

# k8s/networking/ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: app-ingress
  namespace: app
  annotations:
    kubernetes.io/ingress.class: "gce"
    kubernetes.io/ingress.global-static-ip-name: "app-ip"
    # Will add these after SSL setup:
    # networking.gke.io/managed-certificates: "app-certificate"
    # kubernetes.io/ingress.allow-http: "false"
spec:
  defaultBackend:
    service:
      name: react-frontend
      port:
        number: 80
  rules:
  - host: app.example.com  # Replace with your domain
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: dotnet-api
            port:
              number: 8080
      - path: /api/*
        pathType: ImplementationSpecific
        backend:
          service:
            name: dotnet-api
            port:
              number: 8080
      - path: /
        pathType: Prefix
        backend:
          service:
            name: react-frontend
            port:
              number: 80

Deploy Ingress

# Apply BackendConfig
kubectl apply -f k8s/networking/backend-config.yaml

# Update services with annotations
kubectl apply -f k8s/frontend/service.yaml
kubectl apply -f k8s/backend/service.yaml

# Apply Ingress
kubectl apply -f k8s/networking/ingress.yaml

# Watch Ingress status
kubectl get ingress app-ingress -n app -w

# Get Ingress details
kubectl describe ingress app-ingress -n app

Verify Ingress

# Check Ingress has an IP address
kubectl get ingress app-ingress -n app
# NAME          CLASS    HOSTS            ADDRESS        PORTS   AGE
# app-ingress   <none>   app.example.com  34.120.x.x     80      5m

# Get the static IP
gcloud compute addresses describe app-ip --global --format='value(address)'

# Check backend health in GCP Console or CLI
gcloud compute backend-services list --filter="name~k8s"

# Check backend health status
gcloud compute backend-services get-health BACKEND_SERVICE_NAME --global

# Test with curl (HTTP)
curl -H "Host: app.example.com" http://34.120.x.x/
curl -H "Host: app.example.com" http://34.120.x.x/api/health

DNS Configuration

# Get your static IP
STATIC_IP=$(gcloud compute addresses describe app-ip --global --format='value(address)')
echo "Configure DNS A record: app.example.com -> $STATIC_IP"

# Option 1: Using Cloud DNS
gcloud dns record-sets create app.example.com \
    --zone="example-zone" \
    --type="A" \
    --ttl="300" \
    --rrdatas="$STATIC_IP"

# Option 2: Using external DNS provider
# Add an A record in your DNS provider's console:
# Name: app (or @ for root)
# Type: A
# Value: 34.120.x.x (your static IP)
# TTL: 300

Network Endpoint Groups (NEG)

The cloud.google.com/neg annotation enables container-native load balancing. Traffic goes directly to pod IPs instead of through NodePorts, improving performance and enabling better health checking.

Health Check Requirements

The load balancer health check must return HTTP 200. Ensure your /health endpoint exists and returns 200 OK. The health check runs from Google's infrastructure, not from within your cluster.
