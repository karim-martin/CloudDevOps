SSL/HTTPS

Secure your application with Google-managed SSL certificates, HTTPS redirect, and modern TLS configuration.

Google-Managed Certificate

Google automatically provisions and renews Let's Encrypt certificates.

# k8s/networking/managed-certificate.yaml
apiVersion: networking.gke.io/v1
kind: ManagedCertificate
metadata:
  name: app-certificate
  namespace: app
spec:
  domains:
  - app.example.com
  # Add more domains if needed:
  # - www.app.example.com
  # - api.example.com

FrontendConfig (HTTPS Redirect)

Configure SSL policy and HTTP to HTTPS redirect.

# k8s/networking/frontend-config.yaml
apiVersion: networking.gke.io/v1beta1
kind: FrontendConfig
metadata:
  name: app-frontend-config
  namespace: app
spec:
  redirectToHttps:
    enabled: true
    responseCodeName: MOVED_PERMANENTLY_DEFAULT  # 301 redirect
  sslPolicy: app-ssl-policy  # Reference to GCP SSL policy

SSL Policy (GCP)

Create an SSL policy for modern TLS configuration.

# Create SSL policy with modern TLS settings
gcloud compute ssl-policies create app-ssl-policy \
    --profile=MODERN \
    --min-tls-version=1.2

# Verify SSL policy
gcloud compute ssl-policies describe app-ssl-policy

# Alternative: RESTRICTED profile (TLS 1.2+ only, stricter ciphers)
gcloud compute ssl-policies create app-ssl-policy-strict \
    --profile=RESTRICTED \
    --min-tls-version=1.2

SSL Policy Profiles

ProfileMin TLSUse Case
COMPATIBLETLS 1.0Maximum compatibility (legacy clients)
MODERNTLS 1.2Recommended for most applications
RESTRICTEDTLS 1.2PCI-DSS compliance, stricter ciphers
CUSTOMConfigurableFine-grained cipher control

Update Ingress with SSL

# k8s/networking/ingress.yaml (updated)
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: app-ingress
  namespace: app
  annotations:
    kubernetes.io/ingress.class: "gce"
    kubernetes.io/ingress.global-static-ip-name: "app-ip"
    # SSL Configuration
    networking.gke.io/managed-certificates: "app-certificate"
    networking.gke.io/v1beta1.FrontendConfig: "app-frontend-config"
    # Disable HTTP after certificate is active (optional)
    # kubernetes.io/ingress.allow-http: "false"
spec:
  defaultBackend:
    service:
      name: react-frontend
      port:
        number: 80
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: dotnet-api
            port:
              number: 8080
      - path: /
        pathType: Prefix
        backend:
          service:
            name: react-frontend
            port:
              number: 80

Deploy SSL Configuration

# Prerequisites: DNS must be configured first!
# The certificate will only provision if DNS points to the static IP

# Create SSL policy
gcloud compute ssl-policies create app-ssl-policy \
    --profile=MODERN \
    --min-tls-version=1.2

# Apply ManagedCertificate
kubectl apply -f k8s/networking/managed-certificate.yaml

# Apply FrontendConfig
kubectl apply -f k8s/networking/frontend-config.yaml

# Update Ingress with SSL annotations
kubectl apply -f k8s/networking/ingress.yaml

# Watch certificate status
kubectl describe managedcertificate app-certificate -n app

Verify Certificate Status

# Check ManagedCertificate status
kubectl get managedcertificate app-certificate -n app

# Status values:
# - Provisioning: Certificate being created
# - Active: Certificate ready and attached
# - ProvisioningFailed: Check DNS configuration

# Detailed status
kubectl describe managedcertificate app-certificate -n app

# Check in GCP Console
gcloud compute ssl-certificates list

# Test HTTPS (after certificate is Active)
curl -I https://app.example.com

# Test HTTP redirect
curl -I http://app.example.com
# Should return 301 redirect to https://

Self-Managed Certificate (Optional)

For custom certificates or certificates from other CAs.

# Create Kubernetes TLS secret from your certificate files
kubectl create secret tls app-tls-secret \
    --namespace=app \
    --cert=certificate.crt \
    --key=private.key

# Or from files
kubectl create secret tls app-tls-secret \
    --namespace=app \
    --cert=/path/to/fullchain.pem \
    --key=/path/to/privkey.pem
# Ingress with self-managed certificate
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: app-ingress
  namespace: app
  annotations:
    kubernetes.io/ingress.class: "gce"
    kubernetes.io/ingress.global-static-ip-name: "app-ip"
spec:
  tls:
  - hosts:
    - app.example.com
    secretName: app-tls-secret
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: react-frontend
            port:
              number: 80

Troubleshooting

IssueSolution
Certificate stuck in ProvisioningVerify DNS A record points to static IP
ProvisioningFailedCheck domain ownership and DNS propagation
Mixed content warningsEnsure all resources use HTTPS URLs
HTTPS redirect not workingVerify FrontendConfig is applied to Ingress

# Debug certificate provisioning
kubectl describe managedcertificate app-certificate -n app

# Check DNS propagation
dig app.example.com +short
nslookup app.example.com

# Verify DNS points to correct IP
EXPECTED_IP=$(gcloud compute addresses describe app-ip --global --format='value(address)')
ACTUAL_IP=$(dig +short app.example.com)
echo "Expected: $EXPECTED_IP, Actual: $ACTUAL_IP"

# Test SSL certificate
openssl s_client -connect app.example.com:443 -servername app.example.com

# Check certificate details
echo | openssl s_client -connect app.example.com:443 2>/dev/null | openssl x509 -text -noout

# View Ingress events
kubectl describe ingress app-ingress -n app | grep -A 20 Events

Disable HTTP (Optional)

After confirming HTTPS works, optionally disable HTTP entirely.

# Add this annotation to Ingress to disable HTTP
annotations:
  kubernetes.io/ingress.allow-http: "false"
DNS First

Google-managed certificates require DNS to be properly configured BEFORE the certificate will provision. The certificate authority needs to verify domain ownership via HTTP challenge.

Certificate Renewal

Google-managed certificates are automatically renewed before expiration. No action required from you. The certificates are valid for 90 days and renewal starts 30 days before expiry.

Wildcard Certificates

Google-managed certificates do not support wildcards (*.example.com). List each domain explicitly or use self-managed certificates for wildcards.
