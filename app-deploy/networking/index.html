Networking

Expose your GKE services to the internet with Ingress, load balancing, and SSL/TLS certificates for secure HTTPS traffic.

What You'll Configure

Ingress

Configure GKE Ingress with path-based routing to frontend and backend services.

SSL/HTTPS

Set up Google-managed SSL certificates and enforce HTTPS redirect.

Key Components

ComponentPurpose

Prerequisites

# Verify static IP exists (created in Foundation)
gcloud compute addresses describe app-ip --global

# Verify services are running
kubectl get services -n app

# Both services should exist:
# - react-frontend (ClusterIP, port 80)
# - dotnet-api (ClusterIP, port 8080)
DNS Configuration

After deploying the Ingress, update your DNS records to point to the static IP. The ManagedCertificate will only provision after DNS is properly configured.

Provisioning Time

Google-managed certificates can take 15-60 minutes to provision. The Ingress will work with HTTP immediately, but HTTPS requires the certificate to be active.
