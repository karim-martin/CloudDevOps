Artifact Registry

Artifact Registry is GCP's universal package manager for container images, language packages, and OS packages.

Supported Formats

FormatUse Case
DockerContainer images
npmNode.js packages
MavenJava packages
Pythonpip packages
GoGo modules
APT/YUMOS packages

Create Repository

# Create Docker repository
gcloud artifacts repositories create my-repo \
    --repository-format=docker \
    --location=us-central1 \
    --description="Docker images"

# Create npm repository
gcloud artifacts repositories create npm-repo \
    --repository-format=npm \
    --location=us-central1

# List repositories
gcloud artifacts repositories list

Docker Authentication

# Configure Docker to use gcloud credentials
gcloud auth configure-docker us-central1-docker.pkg.dev

# Or for multiple regions
gcloud auth configure-docker \
    us-central1-docker.pkg.dev,us-docker.pkg.dev,asia-docker.pkg.dev

Push & Pull Images

# Image naming format
# LOCATION-docker.pkg.dev/PROJECT_ID/REPOSITORY/IMAGE:TAG

# Tag image
docker tag my-image:latest \
    us-central1-docker.pkg.dev/my-project/my-repo/my-image:latest

# Push image
docker push us-central1-docker.pkg.dev/my-project/my-repo/my-image:latest

# Pull image
docker pull us-central1-docker.pkg.dev/my-project/my-repo/my-image:latest

# List images
gcloud artifacts docker images list \
    us-central1-docker.pkg.dev/my-project/my-repo

# List tags
gcloud artifacts docker tags list \
    us-central1-docker.pkg.dev/my-project/my-repo/my-image

GKE Integration

# GKE automatically authenticates to Artifact Registry in same project
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
spec:
  template:
    spec:
      containers:
      - name: app
        image: us-central1-docker.pkg.dev/my-project/my-repo/my-image:v1.0.0
# For cross-project access, grant GKE service account permission
gcloud artifacts repositories add-iam-policy-binding my-repo \
    --location=us-central1 \
    --member="serviceAccount:PROJECT_NUMBER-compute@developer.gserviceaccount.com" \
    --role="roles/artifactregistry.reader"

Vulnerability Scanning

# Enable scanning (on by default for new repos)
gcloud artifacts repositories update my-repo \
    --location=us-central1 \
    --enable-vulnerability-scanning

# View vulnerabilities
gcloud artifacts docker images list \
    us-central1-docker.pkg.dev/my-project/my-repo \
    --show-occurrences \
    --occurrence-filter='kind="VULNERABILITY"'

# Get detailed scan results
gcloud artifacts docker images describe \
    us-central1-docker.pkg.dev/my-project/my-repo/my-image:latest \
    --show-all-metadata

Cleanup Policies

# Delete untagged images older than 30 days
gcloud artifacts repositories set-cleanup-policies my-repo \
    --location=us-central1 \
    --policy=policy.json
// policy.json
{
  "cleanupPolicies": [
    {
      "id": "delete-old-untagged",
      "action": {"type": "Delete"},
      "condition": {
        "olderThan": "2592000s",
        "tagState": "UNTAGGED"
      }
    },
    {
      "id": "keep-recent-tagged",
      "action": {"type": "Keep"},
      "condition": {
        "tagState": "TAGGED",
        "newerThan": "7776000s"
      },
      "mostRecentVersions": {
        "keepCount": 10
      }
    }
  ]
}

npm Registry

# Configure npm
gcloud artifacts print-settings npm \
    --repository=npm-repo \
    --location=us-central1

# .npmrc
@my-scope:registry=https://us-central1-npm.pkg.dev/my-project/npm-repo/
//us-central1-npm.pkg.dev/my-project/npm-repo/:_authToken=""
//us-central1-npm.pkg.dev/my-project/npm-repo/:always-auth=true

# Authenticate
npx google-artifactregistry-auth

# Publish
npm publish

# Install
npm install @my-scope/my-package

Remote Repositories (Proxy)

# Create remote repo (proxies to Docker Hub)
gcloud artifacts repositories create docker-hub-proxy \
    --repository-format=docker \
    --location=us-central1 \
    --mode=remote-repository \
    --remote-repo-config-desc="Docker Hub" \
    --remote-docker-repo=DOCKER-HUB

# Pull through proxy (caches locally)
docker pull us-central1-docker.pkg.dev/my-project/docker-hub-proxy/nginx:latest

Common Commands

# List repositories
gcloud artifacts repositories list

# Describe repository
gcloud artifacts repositories describe my-repo --location=us-central1

# Delete image
gcloud artifacts docker images delete \
    us-central1-docker.pkg.dev/my-project/my-repo/my-image:v1.0.0

# Delete repository
gcloud artifacts repositories delete my-repo --location=us-central1

# Copy image between repos
gcloud artifacts docker images copy \
    us-central1-docker.pkg.dev/my-project/dev-repo/image:v1 \
    us-central1-docker.pkg.dev/my-project/prod-repo/image:v1
Immutable Tags

Enable immutable tags to prevent overwriting existing image tags. This ensures that :v1.0.0 always refers to the same image.

gcr.io Migration

Container Registry (gcr.io) is deprecated. Migrate to Artifact Registry (pkg.dev) for new features and better integration.
