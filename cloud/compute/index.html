Compute Services

GCP provides multiple compute options for running containerized workloads, from fully managed Kubernetes to serverless containers.

Container Compute Options

Google Kubernetes Engine (GKE)

Managed Kubernetes for complex, scalable container orchestration with full control.

Cloud Run

Serverless containers that scale to zero. Pay only for what you use.

Choosing the Right Service

CriteriaGKECloud Run
ManagementYou manage workloads, Google manages control planeFully managed, serverless
ScalingHPA, VPA, Cluster AutoscalerAutomatic, scales to zero
PricingPay for cluster nodes (always on)Pay per request/CPU time
NetworkingFull Kubernetes networking, custom CNIHTTP/gRPC endpoints
StateStateful workloads supportedStateless only
CustomizationFull K8s API, any workloadLimited to container settings
Best ForComplex microservices, stateful appsAPIs, webhooks, async processing

Hybrid Approach

Many teams use both: GKE for core services requiring complex orchestration, and Cloud Run for event-driven workloads, webhooks, and APIs that benefit from scale-to-zero.
