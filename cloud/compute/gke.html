Google Kubernetes Engine (GKE)

GKE is Google's managed Kubernetes service that handles the control plane, upgrades, and integrations with GCP services.

GKE Modes

FeatureStandardAutopilot
Node managementYou manage node poolsGoogle manages nodes
BillingPay for nodesPay for pod resources
Node customizationFull controlLimited
Security defaultsConfigurableHardened by default
Privileged containersAllowedNot allowed
DaemonSetsAllowedLimited

Create Cluster

Standard Cluster

# Create standard cluster
gcloud container clusters create my-cluster \
    --region=us-central1 \
    --num-nodes=1 \
    --machine-type=e2-medium \
    --enable-ip-alias \
    --enable-autoscaling \
    --min-nodes=1 \
    --max-nodes=5

# Create with more options
gcloud container clusters create my-cluster \
    --region=us-central1 \
    --num-nodes=2 \
    --machine-type=e2-standard-4 \
    --disk-size=50GB \
    --disk-type=pd-standard \
    --enable-ip-alias \
    --enable-autoscaling \
    --min-nodes=1 \
    --max-nodes=10 \
    --enable-autorepair \
    --enable-autoupgrade \
    --workload-pool=PROJECT_ID.svc.id.goog \
    --enable-shielded-nodes \
    --release-channel=regular

Autopilot Cluster

# Create Autopilot cluster (recommended for most workloads)
gcloud container clusters create-auto my-autopilot \
    --region=us-central1 \
    --release-channel=regular

# With private nodes
gcloud container clusters create-auto my-autopilot \
    --region=us-central1 \
    --enable-private-nodes \
    --enable-master-authorized-networks \
    --master-authorized-networks=10.0.0.0/8

Private Cluster

# Create private cluster (no public IPs on nodes)
gcloud container clusters create private-cluster \
    --region=us-central1 \
    --enable-private-nodes \
    --enable-private-endpoint \
    --master-ipv4-cidr=172.16.0.0/28 \
    --enable-ip-alias \
    --enable-master-authorized-networks \
    --master-authorized-networks=10.0.0.0/8 \
    --network=my-vpc \
    --subnetwork=my-subnet \
    --cluster-secondary-range-name=pods \
    --services-secondary-range-name=services

Connect to Cluster

# Get credentials (updates ~/.kube/config)
gcloud container clusters get-credentials my-cluster \
    --region=us-central1

# Verify connection
kubectl cluster-info
kubectl get nodes

Node Pools

# Add node pool
gcloud container node-pools create high-memory \
    --cluster=my-cluster \
    --region=us-central1 \
    --machine-type=e2-highmem-4 \
    --num-nodes=2 \
    --enable-autoscaling \
    --min-nodes=0 \
    --max-nodes=10

# Add GPU node pool
gcloud container node-pools create gpu-pool \
    --cluster=my-cluster \
    --region=us-central1 \
    --machine-type=n1-standard-4 \
    --accelerator=type=nvidia-tesla-t4,count=1 \
    --num-nodes=1

# Add spot/preemptible node pool (cost savings)
gcloud container node-pools create spot-pool \
    --cluster=my-cluster \
    --region=us-central1 \
    --machine-type=e2-standard-4 \
    --spot \
    --num-nodes=3 \
    --enable-autoscaling \
    --min-nodes=0 \
    --max-nodes=20

# List node pools
gcloud container node-pools list --cluster=my-cluster --region=us-central1

# Delete node pool
gcloud container node-pools delete old-pool \
    --cluster=my-cluster \
    --region=us-central1

Workload Identity

# Enable on cluster (if not enabled at creation)
gcloud container clusters update my-cluster \
    --region=us-central1 \
    --workload-pool=PROJECT_ID.svc.id.goog

# Create GCP service account
gcloud iam service-accounts create app-sa

# Grant permissions
gcloud projects add-iam-policy-binding PROJECT_ID \
    --member="serviceAccount:app-sa@PROJECT_ID.iam.gserviceaccount.com" \
    --role="roles/storage.objectViewer"

# Create K8s service account
kubectl create serviceaccount app-ksa -n default

# Bind K8s SA to GCP SA
gcloud iam service-accounts add-iam-policy-binding \
    app-sa@PROJECT_ID.iam.gserviceaccount.com \
    --role="roles/iam.workloadIdentityUser" \
    --member="serviceAccount:PROJECT_ID.svc.id.goog[default/app-ksa]"

# Annotate K8s SA
kubectl annotate serviceaccount app-ksa \
    -n default \
    iam.gke.io/gcp-service-account=app-sa@PROJECT_ID.iam.gserviceaccount.com

Cluster Upgrades

# Check available versions
gcloud container get-server-config --region=us-central1

# Upgrade control plane
gcloud container clusters upgrade my-cluster \
    --region=us-central1 \
    --master \
    --cluster-version=1.28

# Upgrade node pool
gcloud container clusters upgrade my-cluster \
    --region=us-central1 \
    --node-pool=default-pool

# Set release channel
gcloud container clusters update my-cluster \
    --region=us-central1 \
    --release-channel=regular

Maintenance Windows

# Set maintenance window (UTC)
gcloud container clusters update my-cluster \
    --region=us-central1 \
    --maintenance-window-start=2024-01-01T02:00:00Z \
    --maintenance-window-end=2024-01-01T06:00:00Z \
    --maintenance-window-recurrence="FREQ=WEEKLY;BYDAY=SA,SU"

# Set maintenance exclusion (no upgrades during this period)
gcloud container clusters update my-cluster \
    --region=us-central1 \
    --add-maintenance-exclusion-name=holiday \
    --add-maintenance-exclusion-start=2024-12-20T00:00:00Z \
    --add-maintenance-exclusion-end=2025-01-05T00:00:00Z

Common Commands

# List clusters
gcloud container clusters list

# Describe cluster
gcloud container clusters describe my-cluster --region=us-central1

# Resize node pool
gcloud container clusters resize my-cluster \
    --region=us-central1 \
    --node-pool=default-pool \
    --num-nodes=5

# Delete cluster
gcloud container clusters delete my-cluster --region=us-central1
Regional vs Zonal

Regional clusters have 3 control plane replicas across zones for high availability. Zonal clusters have a single control plane. Use regional for production workloads.

Autopilot Limitations

Autopilot doesn't allow privileged containers, host network access, or certain DaemonSets. Use Standard mode if you need these features.
