Database Services

GCP offers managed database services for container workloads - from relational databases to NoSQL and in-memory caching.

Database Options

Cloud SQL

Managed MySQL, PostgreSQL, and SQL Server with automatic backups and HA.

Firestore

Serverless NoSQL document database with real-time sync and offline support.

Memorystore

Managed Redis and Memcached for caching and session storage.

Choosing the Right Database

Use CaseRecommended Service
Traditional relational data, complex queriesCloud SQL (MySQL/PostgreSQL)
Real-time sync, mobile/web appsFirestore
Session storage, cachingMemorystore Redis
Simple key-value cachingMemorystore Memcached
Enterprise SQL Server workloadsCloud SQL for SQL Server

Connecting from Containers

GKE

# Cloud SQL: Use private IP or Cloud SQL Auth Proxy
# Firestore: Use client library with Workload Identity
# Memorystore: Use private IP (VPC only)

Cloud Run

# Cloud SQL: Built-in Unix socket connection
gcloud run deploy my-service \
    --add-cloudsql-instances=PROJECT:REGION:INSTANCE

# Firestore: Use client library (automatic auth)

# Memorystore: Requires VPC connector
gcloud run deploy my-service \
    --vpc-connector=my-connector
Private Connectivity

For production workloads, use private IP addresses. Cloud SQL, Firestore, and Memorystore all support VPC-native connectivity for secure, low-latency access from GKE and Cloud Run.
