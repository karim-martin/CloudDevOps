Memorystore

Memorystore provides managed Redis and Memcached instances for caching, session storage, and real-time analytics.

Redis vs Memcached

FeatureRedisMemcached
Data typesStrings, lists, sets, hashes, sorted setsStrings only
PersistenceYes (RDB, AOF)No
ReplicationYes (Standard tier)No
Pub/SubYesNo
TransactionsYesNo
Use casesSession store, queues, leaderboardsSimple caching

Create Redis Instance

# Basic tier (no replication)
gcloud redis instances create my-redis \
    --region=us-central1 \
    --tier=BASIC \
    --size=1 \
    --redis-version=redis_7_0 \
    --network=projects/PROJECT_ID/global/networks/my-vpc

# Standard tier (with HA)
gcloud redis instances create my-redis-ha \
    --region=us-central1 \
    --tier=STANDARD \
    --size=5 \
    --redis-version=redis_7_0 \
    --network=projects/PROJECT_ID/global/networks/my-vpc \
    --replica-count=1

# Get connection info
gcloud redis instances describe my-redis --region=us-central1 \
    --format='value(host,port)'

Create Memcached Instance

# Create Memcached instance
gcloud memcache instances create my-memcached \
    --region=us-central1 \
    --node-count=3 \
    --node-cpu=1 \
    --node-memory=1GB \
    --network=projects/PROJECT_ID/global/networks/my-vpc

# Get node IPs
gcloud memcache instances describe my-memcached --region=us-central1 \
    --format='value(memcacheNodes[].host)'

# deployment.yaml
apiVersion: apps/v1

      containers:

          value: "10.x.x.x"  # Memorystore private IP
        - name: REDIS_PORT
          value: "6379"

Connect from Cloud Run

# Cloud Run requires VPC connector
gcloud compute networks vpc-access connectors create redis-connector \
    --region=us-central1 \
    --network=my-vpc \
    --range=10.8.0.0/28

# Deploy with VPC connector
gcloud run deploy my-service \
    --image=my-image \
    --vpc-connector=redis-connector \
    --set-env-vars="REDIS_HOST=10.x.x.x,REDIS_PORT=6379"

Node.js (Redis)

const redis = require('redis');

const client = redis.createClient({
  socket: {
    host: process.env.REDIS_HOST,
    port: process.env.REDIS_PORT || 6379
  }
});

await client.connect();

// String operations
await client.set('key', 'value');
await client.set('key', 'value', { EX: 3600 }); // 1 hour TTL
const value = await client.get('key');

// Hash operations
await client.hSet('user:123', { name: 'John', email: 'john@example.com' });
const user = await client.hGetAll('user:123');

// List operations (queue)
await client.lPush('queue', 'task1');
const task = await client.rPop('queue');

// Set operations
await client.sAdd('tags', ['redis', 'cache', 'nosql']);
const tags = await client.sMembers('tags');

// Sorted set (leaderboard)
await client.zAdd('leaderboard', [
  { score: 100, value: 'player1' },
  { score: 200, value: 'player2' }
]);
const top10 = await client.zRange('leaderboard', 0, 9, { REV: true });

// Pub/Sub
const subscriber = client.duplicate();
await subscriber.subscribe('channel', (message) => {
  console.log('Received:', message);
});

await client.publish('channel', 'Hello!');

Python (Redis)

import redis
import os

r = redis.Redis(
    host=os.environ['REDIS_HOST'],
    port=int(os.environ.get('REDIS_PORT', 6379))
)

# String operations
r.set('key', 'value')
r.setex('key', 3600, 'value')  # 1 hour TTL
value = r.get('key')

# Hash operations
r.hset('user:123', mapping={'name': 'John', 'email': 'john@example.com'})
user = r.hgetall('user:123')

# List (queue)
r.lpush('queue', 'task1')
task = r.rpop('queue')

# Set
r.sadd('tags', 'redis', 'cache', 'nosql')
tags = r.smembers('tags')

# Sorted set
r.zadd('leaderboard', {'player1': 100, 'player2': 200})
top10 = r.zrevrange('leaderboard', 0, 9, withscores=True)

C# / .NET (Redis)

using StackExchange.Redis;

var connection = ConnectionMultiplexer.Connect(
    $"{Environment.GetEnvironmentVariable("REDIS_HOST")}:{Environment.GetEnvironmentVariable("REDIS_PORT")}"
);
var db = connection.GetDatabase();

// String operations
await db.StringSetAsync("key", "value");
await db.StringSetAsync("key", "value", TimeSpan.FromHours(1));
var value = await db.StringGetAsync("key");

// Hash operations
await db.HashSetAsync("user:123", new HashEntry[] {
    new HashEntry("name", "John"),
    new HashEntry("email", "john@example.com")
});
var user = await db.HashGetAllAsync("user:123");

// List (queue)
await db.ListLeftPushAsync("queue", "task1");
var task = await db.ListRightPopAsync("queue");

// Sorted set
await db.SortedSetAddAsync("leaderboard", "player1", 100);
var top10 = await db.SortedSetRangeByRankAsync("leaderboard", 0, 9, Order.Descending);

Caching Patterns

// Cache-aside pattern
async function getUser(userId) {
  // Try cache first
  const cached = await redis.get(`user:${userId}`);
  if (cached) {
    return JSON.parse(cached);
  }

  // Cache miss - get from database
  const user = await db.query('SELECT * FROM users WHERE id = ?', [userId]);

  // Store in cache with TTL
  await redis.set(`user:${userId}`, JSON.stringify(user), { EX: 3600 });

  return user;
}

// Write-through pattern
async function updateUser(userId, data) {
  // Update database
  await db.query('UPDATE users SET ? WHERE id = ?', [data, userId]);

  // Update cache
  await redis.set(`user:${userId}`, JSON.stringify(data), { EX: 3600 });
}

// Cache invalidation
async function deleteUser(userId) {
  await db.query('DELETE FROM users WHERE id = ?', [userId]);
  await redis.del(`user:${userId}`);
}

Common Commands

# List Redis instances
gcloud redis instances list --region=us-central1

# Describe instance
gcloud redis instances describe my-redis --region=us-central1

# Scale instance
gcloud redis instances update my-redis \
    --region=us-central1 \
    --size=5

# Delete instance
gcloud redis instances delete my-redis --region=us-central1

# Import/Export (Standard tier only)
gcloud redis instances export gs://bucket/backup.rdb \
    --instance=my-redis \
    --region=us-central1
VPC Only

Memorystore instances are only accessible from the same VPC. For Cloud Run, you must configure a VPC connector. There's no public IP option.

AUTH Support

Memorystore for Redis supports AUTH for password protection. Enable it during instance creation or update with --auth-enabled.
