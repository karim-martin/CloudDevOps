Load Balancing

GCP Load Balancing distributes traffic across instances globally or regionally.

Types

TypeProtocolScopeGKE Use
Global HTTP(S)HTTP/HTTPSGlobalIngress
Regional ExternalTCP/UDPRegionalLoadBalancer Service
Internal HTTP(S)HTTP/HTTPSRegionalInternal Ingress

GKE Ingress

# Creates Global HTTP(S) Load Balancer
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: my-ingress
  annotations:
    kubernetes.io/ingress.class: "gce"
    kubernetes.io/ingress.global-static-ip-name: "my-ip"
spec:
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: my-service
            port:
              number: 80

SSL Certificates

# Google-managed certificate
gcloud compute ssl-certificates create my-cert \
    --domains=app.example.com \
    --global

# List certificates
gcloud compute ssl-certificates list

Health Checks

# Create health check
gcloud compute health-checks create http my-check \
    --port=8080 \
    --request-path=/health

# List health checks
gcloud compute health-checks list

Common Commands

# List load balancer components
gcloud compute forwarding-rules list
gcloud compute backend-services list
gcloud compute url-maps list

# Check backend health
gcloud compute backend-services get-health BACKEND --global
