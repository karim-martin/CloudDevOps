Monitoring & Logging

Cloud Monitoring

GKE automatically sends metrics to Cloud Monitoring. View them in the GCP Console under Monitoring > Metrics Explorer.

Key Metrics

MetricDescription
kubernetes.io/container/cpu/core_usage_timeCPU usage
kubernetes.io/container/memory/used_bytesMemory usage
kubernetes.io/container/restart_countContainer restarts
kubernetes.io/pod/network/received_bytes_countNetwork received
kubernetes.io/node/cpu/allocatable_utilizationNode CPU utilization

Cloud Logging

# View logs via gcloud
gcloud logging read 'resource.type="k8s_container"' --limit=10

# Filter by pod
gcloud logging read 'resource.type="k8s_container" AND resource.labels.pod_name="myapp-xxx"'

# Filter by namespace
gcloud logging read 'resource.type="k8s_container" AND resource.labels.namespace_name="production"'

Structured Logging

Output logs as JSON for better parsing:

{"severity":"INFO","message":"User logged in","userId":"123","timestamp":"2024-01-15T10:30:00Z"}

Alerting

# Create alerting policy via gcloud
gcloud alpha monitoring policies create \
    --display-name="High CPU Alert" \
    --condition-display-name="CPU > 80%" \
    --condition-filter='resource.type="k8s_container" AND metric.type="kubernetes.io/container/cpu/limit_utilization"' \
    --condition-threshold-value=0.8 \
    --condition-threshold-duration=300s \
    --notification-channels=CHANNEL_ID

GKE Dashboard

Access built-in dashboards:

  1. Go to Cloud Console > Monitoring > Dashboards
  2. Select "GKE" from the dashboard list
  3. View cluster, workload, and pod metrics

Debugging with kubectl

CommandDescription
kubectl logs pod-nameView pod logs
kubectl logs -f pod-nameFollow logs
kubectl logs pod-name -c containerSpecific container
kubectl logs --previous pod-namePrevious container logs
kubectl describe pod pod-namePod events
kubectl top podsResource usage
kubectl top nodesNode resource usage
kubectl get events --sort-by='.lastTimestamp'Recent events

Custom Metrics

# Export custom metrics with Prometheus format
# Then use Cloud Monitoring Prometheus sidecar

apiVersion: v1
kind: Pod
metadata:
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/port: "8080"
    prometheus.io/path: "/metrics"
spec:
  containers:
  - name: app
    image: myapp
    ports:
    - containerPort: 8080

Uptime Checks

# Create uptime check
gcloud monitoring uptime-checks create http \
    --display-name="My App Health Check" \
    --uri="https://app.example.com/health" \
    --check-interval=60s \
    --timeout=10s
Observability Stack

For advanced observability, consider using Prometheus + Grafana deployed via Helm, or Google Cloud Managed Service for Prometheus.
